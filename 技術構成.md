# プロジェクト技術構成ドキュメント

## 📋 プロジェクト概要

このプロジェクトは**結核治療支援アプリケーション**です。患者と医療スタッフのコミュニケーション、AI診断支援、学習コンテンツ提供などの機能を提供します。

---

## 🎯 主要機能

- **AIチャット機能**: OpenAI APIを使用した24時間対応のAI医師チャット
- **リアルタイムチャット**: 医師と患者間のリアルタイムコミュニケーション
- **クイズ・Q&A機能**: 学習と理解度確認のためのクイズシステム
- **動画視聴機能**: 教育コンテンツの動画視聴
- **多言語対応**: 日本語、英語、中国語、韓国語に対応
- **プロフィール管理**: ユーザー・医師プロフィール管理
- **進捗管理**: 学習進捗と治療記録の管理

---

## 🖥️ フロントエンド技術

### HTML5 / CSS3
- **用途**: マークアップとスタイリング
- **特徴**: 
  - レスポンシブデザイン対応
  - モバイルファースト設計
  - セマンティックHTML構造

### JavaScript（バニラJS）
- **用途**: 
  - フロントエンドのインタラクティブ機能
  - Firebase SDK連携
  - DOM操作とイベント処理
  - API通信（fetch API）
- **特徴**: 
  - フレームワーク不使用（軽量）
  - ES6+構文使用
  - モジュール化された構造

### Progressive Web App (PWA)
- **用途**: 
  - オフライン対応
  - アプリライクな体験提供
  - プッシュ通知対応（将来実装可能）
- **構成ファイル**:
  - `manifest.json`: アプリマニフェスト設定
  - `sw.js`: Service Worker（オフラインキャッシュ管理）

### 多言語対応システム
- **対応言語**: 
  - 日本語 (ja)
  - 英語 (en)
  - 中国語 (zh)
  - 韓国語 (ko)
- **実装方式**:
  - JSON形式の翻訳ファイル（`lang/*.json`）
  - `data-translate`属性による自動翻訳
  - Firebaseにユーザー言語設定を保存

### 外部ライブラリ・API

#### Font Awesome 6.4.0
- **用途**: アイコン表示
- **読み込み方式**: CDN経由

#### Google Fonts（Noto Sans JP）
- **用途**: 日本語フォント提供
- **特徴**: 可読性の高い日本語表示

#### YouTube IFrame API
- **用途**: 動画視聴機能
- **機能**: 
  - 動画の埋め込み表示
  - 視聴完了検知
  - 動画コントロール

## ⚙️ バックエンド技術

### Python（Flask）
- **用途**: 
  - Webサーバー（`app.py`）
  - WebSocket通信（`server.py`）
  - テンプレートレンダリング
- **主要ライブラリ**:
  - **Flask 3.1.0**: Webフレームワーク
  - **Flask-SocketIO**: WebSocket通信
  - **Flask-CORS**: クロスオリジンリクエスト対応
- **機能**:
  - リアルタイムチャットのWebSocketサーバー
  - ルーティング設定

### Node.js（Express）
- **用途**: 
  - RESTful APIサーバー（`templates/server.js`）
  - OpenAI APIプロキシ
  - 静的ファイル配信
- **主要ライブラリ**:
  - **Express 5.2.1**: Webフレームワーク
  - **node-fetch 3.3.2**: HTTPリクエスト（OpenAI API連携）
  - **dotenv 17.2.3**: 環境変数管理
- **APIエンドポイント**:
  - `POST /api/chat`: AIチャット機能
  - `POST /api/translate`: 翻訳機能

### Rust
- **用途**: 
  - WebSocketチャットサーバー（`templates/src/main.rs`）
  - 高性能なリアルタイム通信
- **特徴**: 
  - メモリ安全性
  - 高パフォーマンス
  - コンパイル済みバイナリ

## 🔥 Firebase サービス構成

### Firebase Hosting
- **用途**: 静的ファイルのホスティング
- **設定**: `firebase.json`で設定
- **特徴**: 
  - CDN配信
  - HTTPS強制
  - カスタムドメイン対応

### Firebase Authentication
- **用途**: ユーザー認証システム
- **対応方式**: 
  - メール/パスワード認証
  - Firebase Auth SDK v8.10.1 / v9.22.1使用
- **機能**:
  - ユーザー登録・ログイン
  - セッション管理
  - 認証状態の監視

### Cloud Firestore
- **用途**: NoSQLデータベース
- **保存データ**:
  - ユーザープロフィール
  - チャットメッセージ
  - クイズ結果
  - 学習進捗
  - 言語設定
- **セキュリティ**: `firestore.rules`でルール定義

### Firebase Functions
- **用途**: サーバーレス関数
- **実装言語**:
  - **TypeScript** (Node.js 22):
    - `functions/src/index.ts`
    - `functions/src/genkit-sample.ts`
    - Firebase Functions v6.0.1
  - **Python**:
    - `functions/main.py`
    - `functions/geminiChat.js`
    - Functions Framework使用
- **ビルド**: TypeScriptは`lib/`にコンパイル

### Firebase Data Connect
- **用途**: GraphQL API + PostgreSQLデータベース
- **構成**:
  - `dataconnect/schema/schema.gql`: GraphQLスキーマ定義
  - `dataconnect/connector/queries.gql`: クエリ定義
  - `dataconnect/connector/mutations.gql`: ミューテーション定義
- **データベース**: Cloud SQL (PostgreSQL)
- **インスタンス**: `pre-fdc`

### Firebase Storage
- **用途**: ファイルストレージ
- **保存対象**: 
  - ユーザーアバター画像
  - 医療関連ドキュメント
- **セキュリティ**: `storage.rules`でルール定義

### Firebase Remote Config
- **用途**: アプリ設定のリモート管理
- **設定ファイル**: `remoteconfig.template.json`

### Firebase App Hosting
- **用途**: フルスタックアプリのホスティング
- **設定**: `apphosting.yaml`
- **特徴**: 
  - Cloud Runベース
  - 自動スケーリング
  - バックエンドとフロントエンドの統合デプロイ

## 🔄 データフロー

1. **ユーザー認証**
   - Firebase Authentication → Firestore（ユーザー情報保存）

2. **AIチャット**
   - フロントエンド → Express API → OpenAI API → レスポンス

3. **リアルタイムチャット**
   - フロントエンド → WebSocket → Firestore → リアルタイム更新

4. **多言語対応**
   - ユーザー選択 → Firestore保存 → ページ読み込み時取得 → 翻訳実行

5. **クイズ機能**
   - OpenAI API → クイズ生成 → Firestore保存 → フロントエンド表示
